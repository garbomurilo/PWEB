# Tema: APIs e integração - Conceitos, Tipos, Ferramentas e exemplos

# Conceito

Uma API (acrônimo para Application Programming Interface ou Interface de Programação de Aplicações) é um conjunto de padrões, normas, ferramentas
e protocolos que permite a integração e a comunicação entre diferentes softwares e seus componentes. De forma simplificada, ela funciona como uma ponte ou intermediária,
garantindo que sistemas desenvolvidos em linguagens distintas possam trocar dados, recursos e funcionalidades de maneira ágil e segura

# Tipos de APIs

  APIs Públicas (ou Abertas)
  Estão disponíveis para qualquer desenvolvedor ou empresa. 
  Podem ser totalmente gratuitas ou exigir algum tipo de autenticação e pagamento conforme o uso. 
  Exemplos comuns incluem a API do Twitter e a do Google Maps.

  APIs Privadas (ou Internas)
  São restritas ao uso interno de uma organização. 
  Elas servem para integrar sistemas e melhorar a comunicação entre diferentes equipes de desenvolvimento dentro da mesma empresa,
  permanecendo ocultas para usuários externos.

  APIs de Parceiros 
  São acessíveis apenas por parceiros de negócios específicos que possuem autorização. 
  Empresas como Uber e Lyft utilizam essa política para permitir que desenvolvedores selecionados integrem seus serviços de transporte em outros aplicativos

  APIs Compostas: 
  Combinam dados ou funcionalidades de várias APIs diferentes em um único endpoint. 
  Elas permitem que um programador acesse diversos recursos com apenas uma chamada, o que é muito útil em arquiteturas de microsserviços.

# Ferramenta

As ferramentas de API são softwares, plataformas ou bibliotecas projetadas para auxiliar desenvolvedores e equipes de garantia de qualidade (QA)
em todas as etapas do ciclo de vida de uma interface, abrangendo desde o planejamento e design até o teste, monitoramento, segurança e comercialização

  1. Amazon API Gateway: É um serviço totalmente gerenciado que permite aos desenvolvedores criar, publicar, manter, monitorar e proteger APIs em qualquer escala. 
  Ele é capaz de processar milhares de chamadas simultâneas, lidando com a gestão de tráfego, autorização e controle de acesso.
  
  2. RapidAPI: Considerado o maior marketplace global de APIs, esta plataforma permite que desenvolvedores descubram e se conectem a mais de 10.000 APIs públicas.
  Ela oferece a conveniência de permitir que os usuários testem as interfaces diretamente na plataforma antes de realizar qualquer integração ou compra.

  3. OpenAPI (anteriormente Swagger): É uma especificação técnica padrão utilizada para o planejamento e design de APIs de alta qualidade. 
  Ela fornece o esquema (blueprint) necessário para que o desenvolvimento siga padrões atuais, facilitando também a geração de documentação interativa e testável.
